# Module 4

## Client Server

Learned that the client server model is the foundation of web communication, where clients such as browsers send requests and servers process those requests and return the required resources.

## Protocol

Understood that protocols are standardized rules that define how devices communicate over a network, including how requests, responses, and errors should be formatted and handled.

## Port

Learned that ports act as communication endpoints on a system, allowing multiple network services to run simultaneously and helping clients connect to the correct service.

## DNS

Studied how DNS translates human readable domain names into IP addresses, enabling users to access websites without remembering numerical addresses.

## HTTP(S)

Learned that HTTP is a stateless protocol used for communication between clients and web servers, while HTTPS secures this communication using encryption. Also explored commonly used HTTP methods such as GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS, CONNECT, and TRACE.

## Virtualisation Basics

Learned that virtualization enables multiple virtual machines to run on a single physical system, with a hypervisor managing resources, isolation, and the lifecycle of virtual machines.

## Cloud Computing Basics

Understood that cloud computing provides on demand access to computing resources such as servers, storage, and applications over the internet without requiring physical infrastructure management.

## Types of Cloud

Studied public, private, and hybrid cloud deployment models and understood how organizations choose different models based on scalability, control, security, and business requirements.

## Cloud Service Models

Learned the differences between IaaS, PaaS, and SaaS service models and understood how management responsibilities are shared between users and cloud providers in each model.
