# Module 3 | How the Web Works

## DNS in Detail

Learned that DNS translates domain names into IP addresses, making it easier for users to access websites without remembering numerical addresses. Also understood the structure of domain names and the role of TLDs.

## DNS Record Types

Studied different DNS records such as A, AAAA, CNAME, MX, and TXT records and learned how each record is used in domain and email management.

## How DNS Resolution Works

Understood how DNS resolution works by following the request flow from local cache to recursive resolver, root server, TLD server, and authoritative server. Also learned about DNS caching and TTL values.

## HTTP in Detail

Learned that HTTP is the protocol used for communication between browsers and web servers, while HTTPS provides secure communication through encryption.

## What a URL Is

Explored the different components of a URL, including scheme, host, port, path, query string, and fragment.

## Making a Request

Learned how browsers send HTTP requests to servers and understood the basic structure of an HTTP request along with the information carried in request headers.

## HTTP Methods

Studied commonly used HTTP methods such as GET, POST, PUT, and DELETE and understood their purpose in web communication.

## HTTP Headers

Learned how HTTP headers provide additional information between clients and servers and explored commonly used request and response headers.

## Cookies

Understood how cookies are used to maintain user sessions and store information because HTTP itself does not remember previous requests.

## How Websites Work

Learned how web servers process client requests and send responses back to browsers. Also understood the difference between frontend and backend components.

## HTML Injection

Studied how improper handling of user input can lead to HTML injection vulnerabilities and learned the importance of input validation and sanitization.
